# Course Guide

This course is a structured path for learning modern AI engineering, with a special focus on LLM applications, agentic systems, tool use, MCP, evaluation harnesses, and production deployment.

The course is designed for serious self-study. Treat it like a textbook, lab manual, and portfolio plan in one repository.

## Who This Is For

This course is for learners who want to become practical AI engineers, not only prompt users. You should be comfortable with basic programming and willing to build projects while learning.

Recommended prerequisites:

- Basic Python
- Basic Git and GitHub
- Comfort reading documentation
- Basic command line usage
- Curiosity about how software systems fail

## Learning Outcomes

By the end of the course, you should be able to:

- Explain how LLM-powered systems differ from deterministic applications.
- Build LLM applications with structured inputs and outputs.
- Design prompts, tools, memory, RAG, and agent loops as maintainable system components.
- Build single-agent and multi-agent systems.
- Use agent frameworks with judgment instead of treating them as magic.
- Design evaluation harnesses for model and agent behavior.
- Integrate MCP servers and clients into agent workflows.
- Deploy, monitor, and improve production AI systems.
- Present a portfolio of projects on GitHub.

## Suggested 6-Week Path

| Week | Theme | Main Output |
|---|---|---|
| 1 | Foundations, setup, first LLM apps, first agent loop | Digital twin assistant prototype |
| 2 | OpenAI platform, structured outputs, tools, guardrails | Deep research assistant |
| 3 | CrewAI and role-based multi-agent systems | Research or analysis crew |
| 4 | LangChain, LangGraph, long-running agents, human review | Sidekick-style agent |
| 5 | Framework landscape, Google ADK, Microsoft, Pydantic AI, TypeScript agents | Multi-framework comparison project |
| 6 | MCP, harness engineering, capstone architecture, deployment | Capstone agent system |

## Study Rhythm

For each chapter:

1. Read the chapter overview.
2. Read the theory.
3. Build the lab.
4. Complete the exercises before reading the solutions.
5. Answer the interview questions out loud.
6. Add notes to your own learning journal.
7. Commit your work.

## Portfolio Rule

Every project should include:

- A short `README.md`
- A design note
- Setup instructions
- Example inputs and outputs
- Tests or evaluation cases
- Known limitations
- Future improvements

The goal is to show engineering thinking, not only working demos.
