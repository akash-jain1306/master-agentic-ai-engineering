# Labs

## Lab 1: Create an AI Feature Design Brief

Create a design brief for a small AI feature before writing code. This lab trains the habit of defining behavior, risks, and evaluation criteria early.

### Scenario

You are building an AI assistant that summarizes meeting notes and extracts action items.

### Deliverable

Create `labs/01-foundations/meeting-notes-brief.md` with these sections:

- User outcome
- Inputs
- Outputs
- Model responsibilities
- Deterministic software responsibilities
- Failure cases
- Evaluation examples
- Logging and monitoring

### Acceptance Criteria

- The output format is explicit.
- At least five failure cases are listed.
- At least three evaluation examples are included.
- The brief separates model behavior from ordinary application behavior.
