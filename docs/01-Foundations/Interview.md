# Interview Preparation

## Core Questions

1. Why are AI agents considered the next step beyond chatbots?
2. What changed after ChatGPT from a software engineering perspective?
3. Explain the difference between AI, ML, deep learning, LLMs, and agents.
4. What is the difference between an LLM application and an AI agent?
5. Why is the LLM only one component of a production AI system?
6. What is context in an LLM application?
7. What is memory in an AI system, and how is it different from context?
8. What problems do tools solve for LLMs?
9. What is an agent loop?
10. What is the difference between a workflow and an agent?

## System Design Questions

1. Design a high-level architecture for an AI coding assistant.
2. Where would you add human approval in an AI email agent?
3. How would you decide whether a task should be deterministic code, an LLM call, a copilot, or an agent?
4. What production metrics would you track for an AI support assistant?
5. How would you explain MCP to a software engineer who has never used it?
6. Where do LangGraph, CrewAI, and OpenAI Agents SDK fit in the stack?
7. How would you prevent an agent from running forever?
8. What should be logged in a tool-using agent?
9. How would you evaluate a research agent?
10. What are the risks of giving an agent browser access?

## Senior-Level Questions

1. Your company wants to add agents to every internal workflow. What architecture principles would you establish first?
2. A team says their agent works well because the demo was impressive. How would you assess production readiness?
3. How would you design an evaluation harness for a coding assistant?
4. What are the tradeoffs between using a framework and building an agent loop yourself?
5. How would you contain the blast radius of bad model output?
6. When would you choose a smaller model over a larger model?
7. How would you compare OpenAI, Anthropic, Google, and local models for a production use case?
8. What should be handled by code instead of the model?
9. How would you design observability for a multi-agent system?
10. What makes AI engineering different from traditional ML engineering?

## Answering Strategy

Strong answers should:

- Separate model capability from system design.
- Mention context, tools, memory, evals, and observability.
- Discuss constraints and tradeoffs.
- Avoid claiming that agents are magic or fully autonomous by default.
- Use concrete examples from real products.
